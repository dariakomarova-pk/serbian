# Логика раздела «Слова»
> Документ для разработки. Версия 1.0.

---

## Навигация

```
/recnik                        — список категорий
/recnik/[categoryId]           — слова категории + выбор режима
/recnik/[categoryId]/flashcard — режим флэшкарт
/recnik/[categoryId]/match     — режим «найди пару»
```

---

## Экран 1 — список категорий `/recnik`

Показывает все категории словаря в виде списка.

**Каждая карточка категории содержит:**
- Emoji категории
- Название на русском
- Количество слов

**Источник данных:** `/data/vocabulary/a1-words.json` — группировка по полю `category`.

---

## Экран 2 — категория `/recnik/[categoryId]`

**Вверху страницы:**
- Кнопка назад
- Название категории (emoji + название)
- Две кнопки режима: `Флэшкарты` и `Найди пару`

**Ниже — список всех слов категории.** Каждое слово:
```
káfa
[ка́фа]
кофе ☕
```

Поля: `word` (с ударением), `transcription` в скобках, `translation`, `emoji`.

---

## Экран 3 — флэшкарты `/recnik/[categoryId]/flashcard`

Пользователь листает карточки одну за одной.

**Одна карточка:**
- Лицо: сербское слово с ударением (`káfa`)
- Транскрипция: `[ка́фа]`
- Перевод и emoji: `кофе ☕`

**Управление:**
- Свайп влево / кнопка «Дальше» → следующая карточка
- Счётчик: `3 / 15`
- По завершении всех карточек — экран «Готово» с кнопкой «Повторить» или «Попробовать Найди пару»

**Логика:** все карточки показываются по порядку, без оценки знания. Простой просмотр.

---

## Экран 4 — найди пару `/recnik/[categoryId]/match`

Классический режим matching.

**Механика:**
- На экране 6 слов на сербском и 6 переводов на русском — перемешанные, в два столбца
- Пользователь нажимает сербское слово → оно выделяется → нажимает русский перевод
- Если пара верная — оба слова исчезают (или зеленеют и исчезают)
- Если неверная — оба на секунду краснеют, сбрасываются
- Когда все 6 пар найдены — экран победы: время + кнопка «Ещё раунд» (следующие 6 слов)

**Логика выбора слов для раунда:**
- Берём слова категории, разбиваем на группы по 6
- Если слов меньше 6 — берём все
- Порядок групп: последовательный (не случайный), чтобы пользователь проходил все слова

## Структура JSON — одно слово

```json
{
  "id": "kafa",
  "category": "07-restoran",
  "categoryName": "Еда и кафе",
  "word": "káfa",
  "transcription": "ка́фа",
  "translation": "кофе",
  "emoji": "☕",
  "partOfSpeech": "imenica"
}
```

**Поля:**
- `id` — уникальный идентификатор (строка, латиница, без пробелов)
- `category` — id категории, совпадает с id темы курса
- `categoryName` — название категории на русском (для отображения)
- `word` — слово на сербском с ударением (акут над ударной гласной: `á é í ó ú`)
- `transcription` — приблизительная русская транскрипция с ударением
- `translation` — перевод на русский
- `emoji` — эмодзи для визуального контекста (опционально, можно `""`)
- `partOfSpeech` — часть речи: `imenica` (сущ.), `glagol` (глагол), `pridev` (прил.), `prilog` (нареч.)

## Технические заметки

- Ударение обозначается акутом прямо в строке слова: `á é í ó ú` — рендерится в браузере без дополнительных библиотек
- Транскрипция — приблизительная, для русскоязычных. Не претендует на точность для сложных звуков (ć, đ)
- Прогресс пользователя (какие слова просмотрены) — в `localStorage`, ключ `vocab_seen_{categoryId}`
- Режим «найди пару»: состояние раунда хранится в React state, не в localStorage
